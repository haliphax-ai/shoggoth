# 2026-05-05

## Fix: Markdown table formatting for builtin-message tool

**Issue:** The `builtin-message` tool was posting Markdown tables directly to Discord without converting them to ASCII box-drawing format. The `mdTableToAscii` conversion was only applied in `sendBody` (used by agent responses), not in the message tool handler.

**Root Cause:** The Discord plugin's `executeMessageToolAction` was passing the raw Discord REST transport directly, bypassing the table formatting that only lived in `DiscordPlatformAdapter.sendBody`.

**Fix:** In `packages/platform-discord/src/plugin.ts`, wrapped the Discord REST transport passed to `executeMessageToolAction` to apply `mdTableToAscii` formatting to:

- `createMessage` - applies formatting to `body.content`
- `createMessageWithFiles` - applies formatting to `body.content`
- `editMessage` - applies formatting to `body.content`

All other transport methods pass through unchanged to the underlying `discordRestTransport`.

**Files Changed:**

- `packages/platform-discord/src/plugin.ts` - Added import for `mdTableToAscii`, wrapped transport in message tool context

**Verification:**

- All 162 tests pass in platform-discord package
- Table formatter tests confirm `inlineCode` and `html` handling works correctly
- TypeScript compilation passes
- The fix ensures `builtin-message` posts via Discord now go through the same table formatting as agent responses

**Note:** This is Discord-specific formatting. Other platforms may handle Markdown tables differently, so this logic belongs in the Discord plugin, not in a shared layer. ⚜️
